**Critique of the current first-shot dashboard mockup and mandatory improvements:**

The attached first dashboard mockup is a decent starting point (clean dark theme, clear bottom nav, quick actions grid, status + metrics row), but it has several serious shortcomings that make it feel unfinished, generic, and not truly representative of an OpenClaw power-user cockpit:

Critique / What's missing or weak:
- Connection state is vague and passive ("Not Connected" + gear icon only) → no clear path to connect/pair, no progress feedback, no security indicators, no Tailscale/Bonjour hints.
- Metrics row (Active / Pending / Done = 0/0/0) lacks explanation → what exactly do these numbers represent? Sessions? Tasks? Memory items? Cron jobs? No tooltips, no color coding beyond basic green/orange.
- Quick Actions tiles are truncated ("Summariz...", "Check Git..."), feel placeholder-level, not customizable, and miss many high-frequency OpenClaw commands.
- No proactive / agent-initiated content → OpenClaw's biggest strength (heartbeats, cron reminders, incoming alerts) is completely absent (no banners, no notification previews, no "Incoming: summarize inbox?" teaser).
- Recent Activity is extremely basic → single line with almost no context/snippet/preview, no session type icon, no quick jump to full chat/memory.
- Zero lobster personality or branding flair → no claw motifs, no subtle animations, no fun micro-interactions (e.g., claw swipe on task complete).
- No visual hierarchy or depth → flat cards, no shadows/gradients/layers, no scrollable sections if content grows, no safe-area/notch handling visible.
- Core visualizations (Kanban, CRM, PKM, Calendar) are invisible → no teasers, previews, or widgets that hint at the real power (e.g., mini Kanban, contact timeline snippet, calendar agenda card, note excerpt).
- No voice input affordance, no haptic feedback hints, no offline/queued state indicator.
- Bottom nav icons are too generic → should feel thematic (e.g., claw silhouette for Dashboard, speech bubble with claw for Chat, checklist pin for Tasks).

**Mandatory emphasis on core visualizations from user vision + visual enhancements:**
The app **must** deeply support, visualize, and make glanceable the following — they are non-negotiable foundations, not stretch goals:

- **Kanban boards** — Primary task visualization. Live-synced columns (To Do / In Progress / Done / Deferred / Archived + custom) from OpenClaw memory, cron tasks, extracted action items, or skills (Linear, Trello/Basecamp Fizzy, Notion, Markdown files). Full drag-and-drop support that translates gestures into natural-language commands sent back to the agent (e.g., "move 'Dentist appointment' from Deferred to Done"). Show progress bars, assignees, due dates where available.
- **CRM / Relationship views** — Conditional on email/calendar/contact skills: per-contact timeline cards (last interaction, email/meeting history, notes), mini pipeline/Kanban for leads/opportunities, searchable relationship graphs or network view. Pull dynamically from memory + tool outputs.
- **PKM (Personal Knowledge Management) surfaces** — Searchable, tagged, explorable note & memory browser. 2nd-brain style feeds (e.g., "deferred notes to review", weekly memory digest, tag clouds, extracted summaries from conversations/docs/files).
- **Calendars** — Native-feeling monthly/weekly/day views synced from calendar skills (Google/Apple/Outlook/ICS). Highlight conflicts, show proactive suggestions ("reschedule overlapping call?"), allow inline create/edit that triggers agent tool calls.

These must be tied to:
- **Tasks** tab → Kanban (primary) + list fallback, drag-drop, filters, bulk actions
- **Memory** tab → PKM-focused: timeline, search, tags, review queues, note previews
- **Dashboard** → teasers/previews/widgets for Kanban progress, upcoming calendar items, CRM highlights (e.g., "3 contacts need follow-up"), deferred PKM items, proactive banners
- proactive elements everywhere (floating banners, push-style cards, notification previews) e.g. "Deferred tasks reminder: 4 items past due", "Agent heartbeat: system healthy"

**Required visual & UX enhancements for v2:**
- Embrace lobster/dark-lobster branding: navy/black base, orange-red-claw accents (#FF5A36 / #FF8C00 gradients), subtle claw motifs in icons/animations
- Add micro-interactions: claw swipe gesture on cards, ripple on quick actions, loading spinner with claw pinch
- Improve hierarchy: use depth (shadows, elevation), grouping, scrollable sections, hero headers
- Make connection obvious & delightful: animated connect button, success confetti/claw snap on pair
- Add customization: reorder/pin quick actions, toggle widget visibility on dashboard
- Show example rich data in mockups (fake but realistic): Kanban with 2-3 cards, calendar with colored events, CRM contact card with timeline dots, memory snippet with tags

In the feature roadmap:
- MVP must include: enhanced Dashboard with teasers, basic Kanban in Tasks tab, Calendar preview/summary, Memory browser with search, CRM contact list teaser (if skills detected), proactive banner area
- Phase 2: full draggable Kanban, deep CRM pipelines/timelines, advanced PKM tagging/review queues, Kanban customization (columns, sorting, swimlanes)
